#4 - نظام Skills المستقل (تطوير المساعد الذكي الحالي)

## ما هي وسبب إضافتها

تطوير المساعد الذكي العائم الموجود حالياً بإضافة نظام "Skills" مستوحى من نظام Agent Skills في Claude. بدلاً من إعادة كتابة نفس التعليمات في كل محادثة، تُحوِّل Skills هذه التعليمات إلى "حزم جاهزة" يستدعيها المساعد تلقائياً عند الحاجة.

المبدأ الأساسي هو **Progressive Disclosure**: عند إرسال رسالة، يمسح المساعد قائمة المهارات المتاحة، يحدد أيها ذات صلة بالسياق، ثم يحمّل تعليماتها فقط — دون ملء نافذة الـ context بمعلومات غير ضرورية.

كل Skill هي ملف Markdown يحتوي على:
- **YAML Frontmatter**: يصف المهارة والـ tags التي تُفعِّلها تلقائياً
- **جسم التعليمات**: الخطوات والقواعد التفصيلية التي يتبعها المساعد
- **موارد اختيارية**: ملفات JSON أو TXT مرجعية مرتبطة بالمهارة

نظام Skills يُضاف كتبويبة داخل **صفحة الإعدادات الموجودة**، ولا يُنشئ تبويبة رئيسية جديدة في شريط التنقل.

## كيف تبدو النتيجة النهائية

### تبويبة Skills داخل الإعدادات
- تبويبة "المهارات / Skills" تُضاف إلى قائمة تبويبات صفحة الإعدادات الموجودة مع أيقونة مناسبة.
- الصفحة مقسومة إلى **قسمين جانبيين** (split panel):
  - **يسار — مكتبة المهارات**: قائمة بالمهارات المتاحة (مدمجة + مخصصة) في شكل بطاقات أو قائمة مضغوطة، مع أزرار إنشاء ورفع في الأعلى وبطاقة ملخص تعرض عدد المهارات المفعّلة.
  - **يمين — لوحة التحرير**: عند النقر على مهارة، يُعرض محتوى ملف .md الكامل في محرر نصي مع معاينة حية (Raw / Preview). إذا لم تُختر مهارة يُعرض placeholder "اختر مهارة للتعديل أو أنشئ مهارة جديدة".

### مكتبة المهارات (القسم الأيسر)
- بطاقة ملخص في الأعلى: عدد المهارات الكلي، عدد المفعّلة، عدد المخصصة.
- كل بطاقة مهارة تعرض: الاسم، الوصف المختصر، الـ tags كـ chips ملونة، مؤشر النوع (مدمجة / مخصصة)، مفتاح تفعيل/تعطيل.
- النقر على البطاقة يُحمِّل محتوى المهارة في لوحة التحرير.
- زر "رفع مهارة (.md)" في أعلى القائمة.
- زر "إنشاء مهارة جديدة" في أعلى القائمة.

### لوحة التحرير (القسم الأيمن)
- محرر نصي يعرض محتوى ملف .md الكامل للمهارة المختارة.
- تبويبتان: "تحرير" (نص خام) و"معاينة" (Markdown مُعرَّض).
- حقول منفصلة في أعلى اللوحة للاسم والوصف والـ tags والأولوية (1-10).
- قسم Metadata (YAML Frontmatter) يُعرَّض ويُحرَّر بشكل مرئي كحقول منفصلة — لا يضطر المستخدم لكتابة YAML يدوياً.
- قسم Resources: قائمة الملفات المرفقة مع زر رفع ملف جديد (+JSON أو TXT).
- زر "حفظ التغييرات"، زر "حذف المهارة" (للمخصصة فقط)، زر "تصدير كـ .md".

### رفع ملف .md
- يقبل الملف بصيغة frontmatter محددة (تفاصيل أدناه).
- يُحلَّل الملف تلقائياً: يُستخرج الاسم والوصف والـ tags والأولوية من YAML.
- تُعرَّض معاينة للمحتوى قبل الحفظ مع رسالة تأكيد.
- إذا كان الاسم موجوداً مسبقاً: تُعرَّض رسالة تأكيد "هل تريد استبدال المهارة الموجودة؟".

### ربط المهارات بالمساعد الذكي الحالي (gaa-panel)
- عند فتح المساعد الذكي: يمسح تلقائياً قائمة المهارات المفعّلة ويحمّل metadata الخاصة بها (الاسم والـ tags فقط — أقل من 200 رمز).
- عند كل رسالة مُرسَلة: تُشغَّل خوارزمية المطابقة (Tags vs رسالة المستخدم)، وإذا تطابقت مهارة يُحمَّل محتوى تعليماتها الكامل ويُحقَن في System Prompt قبل إرسال الطلب للـ API.
- إذا تطابقت أكثر من مهارة: تُحمَّل مهارة الأولوية الأعلى، والباقي يُهمَل.
- يظهر وسم (badge) صغير في واجهة المساعد "⚡ [اسم المهارة]" عند تفعيل مهارة تلقائياً.

### تفعيل المهارة يدوياً من داخل المحادثة
- زر "⚡ Skills" الموجود في شريط الإدخال في gaa-panel يفتح قائمة منبثقة بالمهارات المفعّلة.
- اختيار مهارة يدوياً يُلغي المطابقة التلقائية لتلك المحادثة ويُطبِّق المهارة المختارة بدلاً منها.
- الوسم يظهر بلون مختلف للتمييز بين التفعيل التلقائي والتفعيل اليدوي.

### المهارات المدمجة الافتراضية
يأتي البرنامج بـ 3 مهارات جاهزة (غير قابلة للحذف، قابلة للتعطيل فقط):

| المهارة | الـ Tags | الوظيفة |
|---|---|---|
| pdf-report-formatter | pdf، تصدير، تقرير | تعليمات تنسيق تقارير PDF |
| product-description-writer | وصف، منتج، كتابة | أسلوب كتابة أوصاف المنتجات الرقمية |
| pricing-guide | تسعير، سعر، ربح | منهجية Van Westendorp وهوامش الربح |

### قواعد التصميم
- تبويبة Skills تتبع نفس تصميم باقي تبويبات الإعدادات الموجودة.
- يتبع كل واجهة المستخدم أنماط التصميم الموجودة: تخطيط RTL، تسميات بالعربية، أرقام عربية غربية (0-9)، دعم الوضع الداكن، تخطيط شبكي بالبطاقات مع نفس اصطلاحات التنسيق.

tags: [pdf, تصدير, تقرير]
priority: 5
type: custom

اكتب هنا تعليماتك بأي أسلوب تريد...

- القاعدة الثانية

## أمثلة

## المهام

1. **تصميم نموذج بيانات Skills** — إنشاء هيكل البيانات في localStorage تحت مفتاح `app-skills`: {id, name, slug, description, tags: [string], content: string, resources: [{name, content}], type: 'builtin'|'custom', enabled: boolean, priority: number, createdAt, updatedAt}. دالة ترحيل للبيانات الموجودة وحقن المهارات المدمجة الثلاث عند أول تشغيل.

2. **بناء Skills Matcher** — إنشاء src/utils/skillsMatcher.js: خوارزمية مطابقة تقارن نص رسالة المستخدم مع tags المهارات المفعّلة. تُعيد المهارة الأعلى تطابقاً وأولوية، أو null إذا لم يكن هناك تطابق.

3. **تعديل منطق المساعد الذكي الحالي** — تعديل assistantAPI.js أو الدالة المسؤولة عن بناء الطلب في gaa-panel: قبل كل إرسال، استدعاء skillsMatcher مع نص الرسالة، وإذا عادت مهارة يُحقَن محتوى تعليماتها في System Prompt بعد الـ context الأساسي للبرنامج.

4. **إنشاء مكوّن SkillCard** — بناء src/components/SkillCard.jsx: بطاقة مضغوطة تعرض اسم المهارة، الوصف، الـ tags كـ chips، مؤشر النوع، مفتاح التفعيل. قابلة للنقر لتحميل المحتوى في اللوحة.

5. **إنشاء مكوّن SkillEditor** — بناء src/components/SkillEditor.jsx: لوحة التحرير الكاملة مع حقول الـ metadata المرئية، المحرر النصي وتبويبة المعاينة، قسم Resources، أزرار الحفظ والحذف والتصدير.

6. **إنشاء مكوّن SkillUploader** — بناء src/components/SkillUploader.jsx: يقبل ملف .md، يحلل الـ YAML frontmatter، يعرض معاينة قبل الحفظ، يتحقق من صحة الصيغة، يتعامل مع حالة الاسم المكرر.

7. **إنشاء تبويبة Skills داخل الإعدادات** — تعديل صفحة الإعدادات الموجودة لإضافة تبويبة "المهارات" تحتوي على المكوّن src/components/SkillsTab.jsx الذي يدمج SkillCard (يسار) و SkillEditor (يمين) في تخطيط split panel مع بطاقة الملخص وأزرار الإنشاء والرفع.

8. **تحديث SkillsPicker في gaa-panel** — تعديل مكوّن SkillsPicker الموجود في المساعد ليجلب المهارات من localStorage بدلاً من أي مصدر سابق، وإضافة منطق التمييز البصري بين التفعيل التلقائي واليدوي في الوسم.

9. **إضافة تنسيق CSS** — تنسيق تبويبة Skills داخل الإعدادات: تخطيط split panel، بطاقات المهارات، لوحة التحرير، chips الـ tags، مفتاح التفعيل، ووسم المهارة النشطة في gaa-panel.
